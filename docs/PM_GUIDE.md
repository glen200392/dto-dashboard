# PM 智能維護指南 | PM Smart Maintenance Guide

## 日常操作流程

### 每週更新 (Weekly Update)

1. **開啟儀表板** — 在瀏覽器中打開 `index.html`
2. **進入編輯模式** — 點擊右上角 `✏️ 編輯模式`
3. **切換到 Action Items 分頁** — 點擊 `📝 全部Action Items`
4. **篩選目標項目** — 使用下拉選單篩選特定項目或負責人
5. **逐項更新**：
   - 點擊每一行的「編輯」按鈕
   - 在右側面板更新：狀態、進度百分比、備註
   - 點擊「儲存」
6. **檢視更新結果** — 切換到 `📊 總覽` 確認 KPI 變化
7. **匯出備份** — 點擊 `📤 匯出 JSON`，以日期命名存檔

### 每月檢視 (Monthly Review)

1. 檢查「狀態分佈」圖表，確認延遲/風險項目比例
2. 檢查「月度完成趨勢」，對比計畫 vs 實際
3. 查看「負責人工作量」，識別過載情況
4. 在「依賴關係」分頁檢查關鍵路徑風險
5. 生成月度報告（匯出 CSV 後在 Excel 中進一步分析）

### 每季董事會報告 (Quarterly Board Report)

1. 匯出當季最新數據 (JSON)
2. 截圖關鍵圖表（總覽、項目進度、甘特圖）
3. 使用「AI 代理人」分頁的活動紀錄作為決策參考
4. 對照 Roadmap 文件中的 Deliverables 進行驗收

## 狀態定義

| 狀態 | 英文 | 說明 | 何時使用 |
|------|------|------|---------|
| 未開始 | Not Started | 尚未啟動 | 工作未安排或等待前置條件 |
| 進行中 | In Progress | 正常執行中 | 工作已啟動且按計畫進行 |
| 已完成 | Completed | 已交付驗收 | 所有 deliverables 已交付且經確認 |
| 延遲 | Delayed | 超出預計時程 | 比原定結束時間晚且無法趕上 |
| 風險中 | At Risk | 可能延遲 | 有已知障礙但仍在努力中 |

## 進度百分比指引

| 進度 | 典型情境 |
|------|---------|
| 0% | 未開始 |
| 10-20% | 剛啟動，完成初步規劃 |
| 30-50% | 執行中，完成部分交付物 |
| 60-80% | 大部分完成，進入測試/審查 |
| 90% | 即將完成，最後修正 |
| 100% | 完全完成並驗收 |

## 資料備份策略

建議的檔案命名規則：
```
dto_dashboard_YYYY-MM-DD.json
dto_dashboard_2026-02-07.json
dto_dashboard_2026-03-01.json
```

建議保存位置：
- 公司 SharePoint/Google Drive 共享資料夾
- 本地備份一份
- 每月至少備份一次

## AI 代理人使用提示

目前 AI 代理人模組為前端展示層。未來整合後：

1. **Jarvis** 會在每日固定時間生成 standup 報告
2. **Tracker** 會在 deadline 前 7 天自動發出提醒
3. **Sentinel** 會偵測異常模式（如連續兩週零進度）
4. **Analyst** 會在每月底自動生成 KPI 分析報告
5. **Scribe** 會自動彙整變更紀錄為會議備忘

## 常見問題

**Q: 儀表板資料會消失嗎？**
A: 儀表板的資料存在 HTML 檔案中，不會自動消失。但建議定期匯出 JSON 備份。

**Q: 可以多人同時編輯嗎？**
A: 目前版本為單人使用。多人協作建議：一人負責更新，更新後分享最新 JSON 給團隊。

**Q: 如何分享給其他人？**
A: 將 `index.html` 檔案 + 最新 `dto_dashboard_*.json` 一起分享即可。接收者打開 HTML 後匯入 JSON。

# PM 智能維護指南 | PM Smart Maintenance Guide

## V3.0 導航架構

### 頁面結構
```
📊 總覽 (#/)              → KPI 卡片 + 4 張圖表 + 快速操作
📋 項目 (#/projects)       → 項目列表（卡片/表格切換）
   └── 項目詳情 (#/projects/:id)  → Action Items + Timeline + Team
       └── Action 詳情 (#/projects/:id/actions/:aid)
📅 時程 (#/timeline)       → 全年甘特圖
🤖 Agent (#/agents)        → Agent 控制中心
   ├── Agent 詳情 (#/agents/:id)      → 設定 + 任務 + 指標 + 歷史
   ├── 工作流 (#/agents/workflows)     → 工作流管理
   └── 活動紀錄 (#/agents/activity)    → 全域活動
👥 團隊 (#/team)           → 團隊工作量
   └── 個人詳情 (#/team/:name)
📝 全部 Actions (#/actions) → 完整表格（篩選 + 排序）
⚙️ 設定 (#/settings)       → 主題 + LLM 設定 + 資料管理
```

## 日常操作流程

### 每週更新 (Weekly Update)

1. **開啟儀表板** — 在瀏覽器中打開 `index.html`
2. **進入項目頁面** — 點擊左側導航欄的 `📋 項目`
3. **選擇項目** — 點擊項目卡片進入詳情頁
4. **編輯 Action Items**：
   - 在 Action Items 分頁中點擊「編輯」按鈕
   - 在右側滑出面板更新：狀態、進度百分比、備註
   - 按 ESC 或點擊面板外區域關閉
   - 點擊「儲存」保存變更
5. **檢視更新結果** — 回到 `📊 總覽` 確認 KPI 變化
6. **匯出備份** — 前往 `⚙️ 設定` → 點擊 `匯出 JSON`

### 快速檢視方法

- **看整體進度** → 點擊 `📊 總覽`
- **看特定項目** → `📋 項目` → 點擊項目卡片
- **看全部待辦** → 點擊 `📝 全部 Actions`（支援多條件篩選）
- **看時程全貌** → 點擊 `📅 時程`（全年甘特圖）
- **看團隊工作量** → 點擊 `👥 團隊`
- **看 AI Agent 狀態** → 點擊 `🤖 Agent`

### 每月檢視 (Monthly Review)

1. 在 `📊 總覽` 檢查 KPI 卡片和圖表趨勢
2. 在 `📋 項目` 逐項檢視延遲/風險項目
3. 在 `👥 團隊` 檢查負責人工作量分配
4. 在 `🤖 Agent` 檢視 AI 代理人活動和指標
5. 前往 `⚙️ 設定` 匯出 JSON 作為月度備份

### 每季董事會報告 (Quarterly Board Report)

1. 匯出當季最新數據 (JSON + CSV)
2. 截圖關鍵頁面（總覽、項目進度、甘特圖）
3. 使用 Agent 活動紀錄 (`#/agents/activity`) 作為決策參考
4. 對照 Roadmap 文件中的 Deliverables 進行驗收

## 編輯面板使用

### 開啟方式
- 在項目詳情頁或全部 Actions 表格中，點擊任一項目的「編輯」按鈕

### 關閉方式
- 按 `ESC` 鍵
- 點擊面板外的灰色遮罩區域
- 點擊面板右上角的 `✕` 按鈕
- 點擊「儲存」後自動關閉

### 可編輯欄位
- 狀態（未開始/進行中/已完成/延遲/風險中）
- 進度百分比（0-100%）
- 開始月份 / 結束月份
- 備註

## 狀態定義

| 狀態 | 英文 | 說明 | 何時使用 |
|------|------|------|---------|
| 未開始 | Not Started | 尚未啟動 | 工作未安排或等待前置條件 |
| 進行中 | In Progress | 正常執行中 | 工作已啟動且按計畫進行 |
| 已完成 | Completed | 已交付驗收 | 所有 deliverables 已交付且經確認 |
| 延遲 | Delayed | 超出預計時程 | 比原定結束時間晚且無法趕上 |
| 風險中 | At Risk | 可能延遲 | 有已知障礙但仍在努力中 |

## 進度百分比指引

| 進度 | 典型情境 |
|------|---------|
| 0% | 未開始 |
| 10-20% | 剛啟動，完成初步規劃 |
| 30-50% | 執行中，完成部分交付物 |
| 60-80% | 大部分完成，進入測試/審查 |
| 90% | 即將完成，最後修正 |
| 100% | 完全完成並驗收 |

## 資料備份策略

建議的檔案命名規則：
```
dto_dashboard_YYYY-MM-DD.json
dto_dashboard_2026-02-08.json
```

建議保存位置：
- 公司 SharePoint/Google Drive 共享資料夾
- 本地備份一份
- 每月至少備份一次

## 常見問題

**Q: 儀表板資料會消失嗎？**
A: 資料存在瀏覽器的 localStorage 中，清除瀏覽器資料會重置。建議定期匯出 JSON 備份。

**Q: 可以多人同時編輯嗎？**
A: 目前版本為單人使用。多人協作建議：一人負責更新，更新後分享最新 JSON 給團隊。

**Q: 如何分享給其他人？**
A: 將 `index.html` 檔案 + 最新 `dto_dashboard_*.json` 一起分享即可。接收者打開 HTML 後在設定頁匯入 JSON。

**Q: 如何設定 LLM 連接？**
A: 前往 `⚙️ 設定` 頁面的「LLM 配置」區塊，選擇 Provider、填入 API Endpoint 和 Key。也可以在每個 Agent 詳情頁單獨設定。

**Q: 如何使用本地模型？**
A: 在 LLM 配置中選擇 Ollama 或 vLLM，填入本地伺服器路徑（如 `http://localhost:11434`）和模型名稱。
